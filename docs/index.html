Developer Reference

Quick start & commands

Get up and running in under two minutes.

1 · Clone & Install
Clone the repo and install dependencies with pnpm.
git clone https://github.com/Suydev/isotope-code.git cd isotope-code npm install
2 · Environment Variables
Create a .env file (or set these in your host's dashboard). Both keys are optional — the app degrades gracefully without them.
# .env GEMINI_API_KEY=your_google_gemini_api_key GROQ_API_KEY=your_groq_api_key PORT=3000 # optional, default 3000 SESSION_SECRET=any_random_string # optional
VariableRequiredDescription
GEMINI_API_KEYOptionalGoogle Gemini API key. Get it free at aistudio.google.com
GROQ_API_KEYOptionalGroq fallback key. Free at console.groq.com
PORTOptionalHTTP port. Defaults to 3000.
SESSION_SECRETOptionalExpress session secret. Auto-generated if unset.
3 · Start the Server
Run the Node.js static file server. It injects API keys and patches the React bundles at startup.
node server.mjs # or with npm: npm start # Expected output: # IsotopeAI running on port 3000 # Gemini API key: configured # Groq API key: configured

Common Issues

Troubleshooting guide

Most issues are resolved in under a minute.

⬜
App shows a blank screen or doesn't load ▼

The most common cause is the server not running, or the browser serving a stale Service Worker cache.

  • Make sure node server.mjs is running and shows "IsotopeAI running on port…"
  • Open DevTools → Application → Service Workers → click Unregister, then hard-refresh (Ctrl+Shift+R / Cmd+Shift+R)
  • If on Replit: ensure the isotope-app workflow is running in the workspace sidebar
  • Check the browser console for any 404 errors on bundle files — the bundle filenames are patched in server.mjs
# restart server node server.mjs
🤖
AI assistant not responding / "AI unavailable" ▼

The AI features require at least one API key. Check startup logs — the server prints which keys are detected.

  • Confirm GEMINI_API_KEY and/or GROQ_API_KEY are set in your environment
  • On Replit: set them in Secrets (padlock icon) — not in a .env file
  • The server injects keys via window.__IK__ — if the page loaded before the key was set, hard-refresh
  • Gemini free tier has rate limits — if you hit them, Groq kicks in automatically
# verify keys are detected at startup: # IsotopeAI running on port 3000 # Gemini API key: configured # Groq API key: configured
🎬
Background video or uploaded wallpaper not showing ▼

The background system uses IndexedDB to persist uploaded media. A few things can block it:

  • Make sure bg-video.js and liquid-glass.css are loading (check Network tab)
  • The default Mustang video is embedded as a Base64 data URI and plays on load — if it's not visible, check that autoplay is not blocked by your browser
  • Uploaded wallpapers go through IndexedDB — if you cleared site data, the upload is gone
  • Video autoplay requires the page to be interacted with first on some browsers (click anywhere then refresh)
  • Confirm the z-index scheme: #root must have z-index: 4, the dim overlay z-index: 3
🪟
Glass / blur effects not visible ▼

backdrop-filter: blur() requires a background behind the element. If the HTML/body are opaque black, glass panels look flat.

  • Confirm liquid-glass.css is loaded — it sets html, body { background: transparent }
  • If you see solid dark panels: toggle the wallpaper preset in Settings → App Wallpaper to Space/Ember/Forest
  • The ◈ glass settings button (bottom-left) lets you adjust blur up to 80px and opacity in real time
  • Firefox has partial backdrop-filter support — enable layout.css.backdrop-filter.enabled in about:config or use Chrome/Safari
🎵
Ambient sounds not playing in the Focus timer ▼

Sounds are served locally from /sounds/ — no internet connection needed. If they're silent:

  • Check that public/sounds/thunder-rain.wav, crickets.wav, and wind.wav exist (each is ~12–17 MB)
  • Browser autoplay policy blocks audio until the user interacts — start a focus session by tapping the timer first
  • Check that server.mjs SOUND_PATCHES replaces the original GitHub raw URLs with /sounds/ paths in the Focus bundle
  • Confirm the patched bundle filename matches FOCUS_BUNDLE in server.mjs
# verify sounds exist ls -lh public/sounds/ # should show: # thunder-rain.wav ~15M # crickets.wav ~12M # wind.wav ~17M
📱
"Add to Home Screen" / PWA install not appearing ▼

PWA install requires HTTPS, a valid manifest, and a registered Service Worker.

  • On Replit: the preview URL is served over HTTPS — install should work. Use Share → Open in new tab first
  • On localhost: Chrome treats localhost as secure. Use http://localhost:3000 and look for the install icon in the address bar
  • On iOS Safari: tap the Share icon → Add to Home Screen
  • Check /manifest.webmanifest returns a valid JSON with start_url and icon paths
  • Open DevTools → Application → Manifest to check for errors
🔁
Onboarding keeps showing on every reload ▼

Onboarding triggers when isotope-auth, isotope-onboarding, and isotope_intro_seen are absent from localStorage.

  • This is intentional — restore-and-launch.js never pre-sets these keys so every fresh install starts with onboarding
  • If you cleared localStorage (or opened in incognito) the onboarding will run again
  • To skip: tap ⚡ Try without signing up at the bottom of the signup page, or use Try Demo on the landing page
# manually skip onboarding in browser console: localStorage.setItem('isotope_intro_seen', 'true') location.reload()
🔌
Port already in use / server won't start ▼

The default port is 3000. If something else is using it, set a different one.

# set a custom port: PORT=8080 node server.mjs # or in .env: PORT=8080 # find what's on port 3000: lsof -i :3000 # macOS / Linux netstat -ano | findstr :3000 # Windows
